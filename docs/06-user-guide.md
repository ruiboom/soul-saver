# SOUL SAVER — User Guide

*Everything you need to install, run, and configure the game. For how to actually play,
see the [Gameplay Guide](07-gameplay-guide.md).*

---

## 1. Requirements

- **macOS** (Apple Silicon or Intel) — primary platform for now.
  Windows and Linux work from the same project via Godot's export dialog.
- **Godot 4.7+** (free, ~100 MB). No other dependencies — all assets ship in the repo.

## 2. Install & run

```sh
# 1. Get Godot (once)
brew install --cask godot

# 2. Get the game
git clone https://github.com/<you>/soul-saver.git
cd soul-saver

# 3. Play
/Applications/Godot.app/Contents/MacOS/Godot --path .
```

Alternatively open the Godot app, click **Import**, choose the `soul-saver` folder, and
press **▶** (Cmd-B). The first launch imports assets (a few seconds); after that it's instant.

To make a double-clickable standalone app: in the Godot editor, *Project → Export…* →
add the **macOS** preset (it offers to download export templates) → **Export Project**.

## 3. Controls

| Action | Keyboard | Gamepad |
|---|---|---|
| Move | **WASD** or arrow keys | Left stick |
| Interact (claim Vestige, enter Gate, advance story text) | **Space** or **E** | **A** / Cross |
| Pause | **Esc** or **P** | Start |
| Menu navigation | Mouse or arrows + Enter | Stick + A |

There is no aim and no fire button — your relics act on their own. Movement is the game.

## 4. Settings & save data

- **Window**: resizable; the game renders at 1920×1080 and scales cleanly.
- **Save file**: `~/Library/Application Support/Godot/app_userdata/Soul Saver/soulsaver_save.json`
  — holds your Ossuary Marks, blessings, Vestige Book and stats. Delete it for a fresh start.
  Saves are versioned; future updates migrate them automatically.
- **Audio**: `music_volume` / `sfx_volume` can be edited in that file (0.0–1.0). An
  in-game options panel is on the roadmap.

## 5. Debug & testing tools (debug builds only)

| Key | Effect |
|---|---|
| `F2` | +200 Grace |
| `F3` | Skip forward 60 seconds |
| `F4` | Spawn a Herald elite |
| `F6` | God mode |
| `F7` | Toggle 4× game speed |

Automated test harness (used in CI/dev — a bot plays the game):

```sh
# 30-second smoke test with screenshots
SOULSAVER_AUTOTEST=1 SOULSAVER_QUIT_AFTER=30 SOULSAVER_SHOT_DIR=/tmp/shots \
  /Applications/Godot.app/Contents/MacOS/Godot --path .

# full 19-minute run at 8x speed: shrines, keepers, Warden, endings
SOULSAVER_AUTOTEST=1 SOULSAVER_QUEST=1 SOULSAVER_TIMESCALE=8 SOULSAVER_QUIT_AFTER=1300 \
  /Applications/Godot.app/Contents/MacOS/Godot --headless --path .
```

## 6. Rebuilding the procedural assets

All sprites are hand-authored SVGs and all audio is synthesized — both generated by
scripts, so the entire asset base is reproducible and license-clean:

```sh
python3 tools/gen_svg.py     # 46 sprites → assets/
python3 tools/gen_audio.py   # 17 sounds → assets/audio/
```

Edit the generators, re-run, and Godot re-imports on next launch.

## 7. Troubleshooting

- **Black window / nothing renders** — make sure you're on Godot 4.7+, not 3.x.
- **"Command not found: godot"** — use the full path `/Applications/Godot.app/Contents/MacOS/Godot`.
- **Choppy on first seconds** — shader compilation warm-up; it settles immediately.
- **`ERROR: resources still in use at exit`** in the terminal — harmless Godot cleanup
  noise when quitting mid-run.
- **Want a fresh meta-progression** — delete the save file (path above).
